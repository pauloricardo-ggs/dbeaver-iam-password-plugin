# DBeaver AWS RDS IAM Auth

Plugin for DBeaver that adds a `PostgreSQL AWS RDS IAM` driver for connecting to
PostgreSQL on AWS RDS using IAM authentication tokens generated by the AWS CLI.

## How It Works

The plugin registers a DBeaver driver backed by:

```text
dev.pauloricardo.dbeaver.aws.rds.iam.driver.AwsRdsIamPostgresDriver
```

Connections using this JDBC prefix are handled by the plugin:

```text
jdbc:awsrdsiam:postgresql:
```

Before each new JDBC connection, the plugin runs:

```bash
aws rds generate-db-auth-token
```

Then it passes the generated token as the PostgreSQL password and delegates the
actual connection to the official PostgreSQL JDBC driver.

## Install In DBeaver

In DBeaver, install from the update site:

```text
Help > Install New Software > Work with: https://pauloricardo-ggs.github.io/dbeaver-iam-password-plugin/
```

Select the AWS RDS IAM auth feature, finish the installation, and restart
DBeaver when prompted.

## Create The Connection

Create a new connection and select the driver:

```text
PostgreSQL AWS RDS IAM
```

Use this JDBC URL format:

```text
jdbc:awsrdsiam:postgresql://{host}:{port}/{database}?sslmode=require
```

Example:

```text
jdbc:awsrdsiam:postgresql://mydb.abc123.us-east-1.rds.amazonaws.com:5432/app?sslmode=require
```

In the authentication section, configure:

| Property | Default |
|---|---|
| `Username` | database user mapped to IAM authentication in RDS |
| `AWS CLI Path` | `aws` |
| `AWS Profile` | `default` |
| `AWS Region` | `us-east-1` |
| `Session Role` | empty (optional, PostgreSQL role applied with `SET ROLE`) |

The connection does not use a stored password. The plugin generates an IAM token
and sends it to PostgreSQL as the password when DBeaver opens the JDBC
connection.

If DBeaver cannot find the AWS CLI, set `AWS CLI Path` to the full binary path
returned by:

```text
macOS: which aws
Windows: where.exe aws
```

Older connections can still pass these values in the JDBC URL query string:

| Query parameter | Default |
|---|---|
| `awsRegion` | `us-east-1` |
| `awsProfile` | `default` |
| `awsCliPath` | `aws` |
| `sessionRole` | empty (optional) |
