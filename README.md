# DBeaver AWS RDS IAM Auth

Plugin for DBeaver that adds a `PostgreSQL AWS RDS IAM` driver for connecting to
PostgreSQL on AWS RDS using IAM authentication tokens generated by the AWS CLI.
It supports direct connections and connections through AWS Systems Manager.

## How It Works

The plugin registers a DBeaver driver backed by:

```text
dev.pauloricardo.dbeaver.aws.rds.iam.driver.AwsRdsIamPostgresDriver
```

Connections using this JDBC prefix are handled by the plugin:

```text
jdbc:awsrdsiam:postgresql:
```

Before each new JDBC connection, the plugin runs:

```bash
aws rds generate-db-auth-token
```

Then it passes the generated token as the PostgreSQL password and delegates the
actual connection to the official PostgreSQL JDBC driver.

The `AWS RDS SSM` authentication model first resolves an EC2 instance and an RDS
DB instance by their `Name` tags. It opens an
`AWS-StartPortForwardingSessionToRemoteHost` session through the EC2 instance,
generates the IAM token for the real RDS endpoint, and connects PostgreSQL to the
configured local port.

## Install In DBeaver

In DBeaver, install from the update site:

```text
Help > Install New Software > Work with: https://pauloricardo-ggs.github.io/dbeaver-iam-password-plugin/
```

Select the AWS RDS IAM auth feature, finish the installation, and restart
DBeaver when prompted.

## Create The Connection

Create a new connection and select the driver:

```text
PostgreSQL AWS RDS IAM
```

Use this JDBC URL format:

```text
jdbc:awsrdsiam:postgresql://{host}:{port}/{database}?sslmode=require
```

Example:

```text
jdbc:awsrdsiam:postgresql://mydb.abc123.us-east-1.rds.amazonaws.com:5432/app?sslmode=require
```

In the authentication section, configure:

| Property | Default |
|---|---|
| `Username` | database user mapped to IAM authentication in RDS |
| `AWS CLI Path` | `aws` |
| `AWS Profile` | `default` |
| `AWS Region` | `us-east-1` |
| `Session Role` | empty (optional, PostgreSQL role applied with `SET ROLE`) |

The connection does not use a stored password. The plugin generates an IAM token
and sends it to PostgreSQL as the password when DBeaver opens the JDBC
connection.

If DBeaver cannot find the AWS CLI, set `AWS CLI Path` to the full binary path
returned by:

```text
macOS: which aws
Windows: where.exe aws
```

Older connections can still pass these values in the JDBC URL query string:

| Query parameter | Default |
|---|---|
| `awsRegion` | `us-east-1` |
| `awsProfile` | `default` |
| `awsCliPath` | `aws` |
| `sessionRole` | empty (optional) |

## Connect Through AWS Systems Manager

Create the same `PostgreSQL AWS RDS IAM` connection and select `AWS RDS SSM` in
the authentication selector. The host in the JDBC URL is ignored for network and
IAM token purposes in this mode; the database name and JDBC query parameters are
still read from the URL.

Configure:

| Property | Description |
|---|---|
| `EC2 Tag` | Value of the EC2 instance's `Name` tag |
| `RDS Tag` | Value of the RDS DB instance's `Name` tag |
| `RDS Port` | Remote database port, normally `5432` |
| `Local Port` | Local port selected by the user for the tunnel |
| `Username` | Database user mapped to IAM authentication |
| `Session Role` | Optional PostgreSQL role applied with `SET ROLE` |
| `AWS CLI Path` | AWS CLI executable, default `aws` |
| `AWS Profile` | AWS CLI profile, default `default` |
| `AWS Region` | Region containing both resources |

Both tags always use the fixed key `Name`. The values must identify exactly one
running EC2 instance and exactly one available PostgreSQL RDS DB instance. The
plugin never falls back to a direct connection when SSM setup fails.

The developer workstation must have the AWS CLI and Session Manager plugin
installed. The selected profile needs permission for `ec2:DescribeInstances`,
`tag:GetResources`, `rds:DescribeDBInstances`, `ssm:StartSession`, and the
existing `rds-db:connect` permission. The EC2 instance must be online in Systems
Manager and able to resolve and reach the RDS endpoint on `RDS Port`.

The initial SSM implementation supports provisioned RDS DB instances (`rds:db`).
Aurora DB clusters are not resolved by this authentication model.
