# DBeaver AWS RDS IAM Auth

Eclipse/DBeaver plugin for connecting to PostgreSQL on AWS RDS with IAM auth
tokens generated by the AWS CLI.

Current MVP: this project ships a JDBC driver wrapper:

```text
br.com.ma9.dbeaver.aws.rds.iam.driver.AwsRdsIamPostgresDriver
```

The wrapper generates a fresh RDS IAM token before each new connection and then
delegates the real connection to the official PostgreSQL JDBC driver.

This avoids depending on DBeaver internal Secret Provider APIs, which can vary
by DBeaver edition and version.

## How It Works

The driver receives a PostgreSQL JDBC URL with this prefix:

```text
jdbc:ma9awsrds:postgresql:
```

It converts the URL to the regular PostgreSQL prefix, generates the password
with:

```bash
aws rds generate-db-auth-token \
  --profile <profile> \
  --hostname <host> \
  --port <port> \
  --region <region> \
  --username <user>
```

Then it calls `org.postgresql.Driver` with `password=<generated-token>`.

## Driver Properties

Configure these as DBeaver driver properties or query params in the JDBC URL.

| Property | Required | Default | Example |
|---|---:|---|---|
| `user` | yes | - | `app_user` |
| `awsRegion` | no | `us-east-1` | `us-east-1` |
| `awsProfile` | no | `default` | `dev` |
| `awsCliPath` | no | `aws` | `/opt/homebrew/bin/aws` |
| `awsRdsIamDebug` | no | `false` | `true` |

Example URL:

```text
jdbc:ma9awsrds:postgresql://mydb.abc123.us-east-1.rds.amazonaws.com:5432/app?sslmode=require&awsRegion=us-east-1&awsProfile=dev
```

## Build

Requirements:

- Java 17+
- Maven 3.9+
- AWS CLI installed for runtime use

```bash
mvn clean verify
```

The P2 update site is generated at:

```text
releng/br.com.ma9.dbeaver.aws.rds.iam.repository/target/repository
```

## Install In DBeaver

Use the published P2 update site:

```text
https://pauloricardo-ggs.github.io/dbeaver-iam-password-plugin/
```

In DBeaver:

```text
Help > Install New Software > Work with: https://pauloricardo-ggs.github.io/dbeaver-iam-password-plugin/
```

Select the AWS RDS IAM auth feature and finish the installation. Restart
DBeaver when prompted.

## Local Development Install

For testing an unpublished local build:

```text
Help > Install New Software > Add > Local
```

Select the generated repository directory:

```text
releng/br.com.ma9.dbeaver.aws.rds.iam.repository/target/repository
```

## Configure DBeaver

Create a custom PostgreSQL driver using:

```text
br.com.ma9.dbeaver.aws.rds.iam.driver.AwsRdsIamPostgresDriver
```

URL template:

```text
jdbc:ma9awsrds:postgresql://{host}:{port}/{database}?sslmode=require&awsRegion=us-east-1&awsProfile=default
```

The IAM token expires after about 15 minutes, so the wrapper generates a new
token for every new JDBC connection.

## Publishing

The GitHub Actions workflow publishes a release when a tag matching `vX.X.X` is
pushed, for example:

```bash
git tag v0.1.0
git push origin v0.1.0
```

For release tags, the pipeline:

- builds the Tycho project;
- creates `dbeaver-aws-rds-iam-auth-update-site.zip`;
- attaches the ZIP to a GitHub Release;
- publishes the P2 update site to the `gh-pages` branch.

In the repository settings, configure GitHub Pages with source `Deploy from a branch`,
branch `gh-pages`, folder `/`.
After the first tag release, users can install from the Pages URL:

```text
Help > Install New Software > Work with: https://pauloricardo-ggs.github.io/dbeaver-iam-password-plugin/
```
